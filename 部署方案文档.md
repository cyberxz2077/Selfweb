## 三、部署流程

### 3.1 自动化部署

**部署流程**：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   GitHub     │────▶│  Vercel      │────▶│  CDN         │
│   Push       │     │  Build       │     │  Deploy      │
└──────────────┘     └──────────────┘     └──────────────┘
```

**详细步骤**：

1. **代码提交**：开发者提交代码到 GitHub
2. **触发构建**：Vercel 监听到 push 事件，自动触发构建
3. **执行构建**：Vercel 执行 `next build` 命令
4. **部署上线**：构建成功后，自动部署到 CDN
5. **缓存刷新**：CDN 缓存自动刷新

### 3.2 部署命令

**开发环境**：

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建项目
npm run build

# 启动生产服务器
npm start
```

**生产环境**：

```bash
# 构建项目
npm run build

# 启动服务器
npm start
```

## 四、CI/CD 配置

### 4.1 GitHub Actions

**文件路径**：`.github/workflows/deploy.yml`

```yaml
name: Deploy to Production

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test
      
      - name: Build project
        run: npm run build
        env:
          NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.SUPABASE_ANON_KEY }}
      
      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v20
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

## 六、性能优化

### 6.1 CDN 优化

**Vercel Edge Cache**：

```json
{
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Cache-Control",
          "value": "public, max-age=31536000, immutable"
        }
      ]
    }
  ]
}
```

**缓存策略**：

| 资源类型 | 缓存时间 | 说明 |
|---------|---------|------|
| 静态资源 | 1 年 | 图片、CSS、JS |
| HTML 页面 | 0 | 每次请求都检查更新 |
| API 数据 | 1 小时 | 根据数据更新频率调整 |

### 6.2 图片优化

**Next.js Image 组件**：

```tsx
import Image from 'next/image';

<Image
  src="/images/projects/claw-os-thumb.png"
  alt="Claw OS"
  width={800}
  height={450}
  priority
  quality={85}
  format="webp"
  loading="eager"
  className="rounded-lg"
/>
```

**优化策略**：

| 策略 | 说明 |
|-----|------|
| **WebP 格式** | 更小的文件体积 |
| **响应式图片** | 根据设备加载不同尺寸 |
| **Lazy Loading** | 懒加载图片 |
| **Blur Hash** | 占位模糊效果 |

```json
{
```

```json
    {
      "source": "/(.*)",
      "headers": [
        {
          "key": "Access-Control-Allow-Origin",
```

### 9.3 SQL 注入防护

**Supabase RLS**：

```sql
-- 启用 RLS
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

-- RLS 策略
CREATE POLICY "允许公开读取" ON projects FOR SELECT USING (true);
CREATE POLICY "允许 authenticated 写入" ON projects FOR ALL USING (auth.role() = 'authenticated');
```

### 9.4 XSS 防护

**Next.js 自动转义**：

```tsx
// Next.js 自动转义
const content = "<script>alert('xss')</script>";
return <div>{content}</div>; // 安全，不会执行
```

**CSP 策略**：

```json
{

    {
```
