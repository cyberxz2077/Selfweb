# 黄鑫哲个人品牌网站 - 技术架构文档 v1.0

## 一、项目概述

### 1.1 项目定位

个人品牌网站，面向公司、合伙人、投资人、粉丝展示个人成长轨迹、现在项目、未来展望。

### 1.2 核心理念

**「极光棱镜」**：将个人成长轨迹（会展→家居→教育→AI→创业）可视化为光的折射，每一次转折都是一次光谱分解，最终汇聚成独特的个人色彩。

### 1.3 目标用户

- 公司 HR / 招聘负责人
- 合伙人 / 投资人
- 粉丝 / 行业同行
- 未来潜在合作方

## 二、技术栈选型

### 2.1 核心框架

| 层级 | 技术选择 | 理由 |
|-----|---------|------|
| **框架** | Next.js 14 (App Router) | SSR/SSG 支持、路由系统、性能优化 |
| **样式** | Tailwind CSS + CSS Modules | 实用优先、定制化、与设计规范完美匹配 |
| **动画** | Framer Motion | React 动画、与 Next.js 无缝集成 |
| **图标** | Lucide React | 简洁几何风格、免费商用 |
| **部署** | Vercel | Next.js 官方部署、自动优化 |
| **数据库** | Supabase | PostgreSQL + Realtime + Storage |

### 2.2 为什么选择 Next.js 14

1. **性能优化**：自动代码分割、图片优化、字体优化
2. **SEO 友好**：SSR/SSG 支持，搜索引擎友好
3. **路由系统**：App Router 提供更好的布局和数据流
4. **开发体验**：自动热重载、TypeScript 支持
5. **部署简单**：Vercel 一键部署

### 2.3 技术栈对比

| 技术 | Next.js | Gatsby | Remix | Vite + React |
|-----|---------|--------|-------|--------------|
| SSR/SSG | ✅ | ✅ | ❌ | ❌ |
| 路由系统 | ✅ App Router | ✅ | ✅ | ❌ |
| 性能优化 | ✅ | ✅ | ❌ | ❌ |
| 开发体验 | ✅ | ❌ | ❌ | ✅ |
| 部署难度 | ✅ | ❌ | ❌ | ❌ |

## 四、Supabase 数据库设计

### 4.1 数据库表结构

#### 4.1.1 projects 表（项目）

```sql
CREATE TABLE projects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(100) NOT NULL,
  slug VARCHAR(100) UNIQUE NOT NULL,
  description TEXT NOT NULL,
  short_description VARCHAR(200),
  type VARCHAR(50) NOT NULL,
  tech_stack TEXT[] NOT NULL,
  role VARCHAR(50) NOT NULL,
  url VARCHAR(500),
  demo_url VARCHAR(500),
  bp_url VARCHAR(500),
  thumbnail_url VARCHAR(500),
  screenshots TEXT[],
  status VARCHAR(20) DEFAULT 'active',
  sort_order INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引
CREATE INDEX idx_projects_slug ON projects(slug);
CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_sort_order ON projects(sort_order);

-- 启用 RLS
ALTER TABLE projects ENABLE ROW LEVEL SECURITY;

-- RLS 策略
CREATE POLICY "允许公开读取" ON projects FOR SELECT USING (true);
CREATE POLICY "允许 authenticated 写入" ON projects FOR ALL USING (auth.role() = 'authenticated');
```

## 五、Obsidian 同步方案

### 5.1 同步架构

```
Obsidian Vault (本地)
         │
         ▼
   同步脚本 (scripts/sync-obsidian.ts)
         │
         ▼
   Supabase Database (远程)
         │
         ▼
   Next.js App (前端)
```

### 5.2 Obsidian 文件结构

```
vault/
├── 项目/                    # 项目文章
│   ├── Claw OS.md
│   ├── Startup Hub.md
│   └── 问记.md
├── 思考/                    # 思考文章
│   ├── 2026-03-02 AI Agent的下一个形态.md
│   └── 2026-03-01 为什么我要做Claw OS.md
├── 关于我/                  # 关于我
│   └── 关于我.md
└── 技能/                    # 技能清单
    ├── AI工具.md
    ├── Agent平台.md
    └── 开发工具链.md
```

### 5.3 同步脚本功能

```typescript
// scripts/sync-obsidian.ts

// 1. 读取 Obsidian Vault
// 2. 解析 Markdown 文件
// 3. 提取 Frontmatter 元数据
// 4. 转换为 Supabase 格式
// 5. 上传到 Supabase
// 6. 生成同步日志

// 支持的文件格式
interface ObsidianFile {
  path: string;
  content: string;
  frontmatter: {
    title: string;
    tags: string[];
    date: string;
    status?: string;
    version?: string;
  };
}

// Supabase 数据格式
interface SupabaseData {
  table: string;
  data: Record<string, unknown>;
}
```

### 5.4 同步流程

```
用户修改 Obsidian
         │
         ▼
   保存到本地
         │
         ▼
   运行同步脚本
         │
         ├─→ 解析文件
         ├─→ 提取元数据
         ├─→ 转换格式
         └─→ 上传 Supabase
         │
         ▼
   Supabase 更新
         │
         ▼
   Next.js 自动重新生成
         │
         ▼
   网站内容更新
```

## 七、性能优化策略

### 7.1 图片优化

```typescript
// 使用 Next.js Image 组件
import Image from 'next/image';

<Image
  src="/images/projects/claw-os.png"
  alt="Claw OS 截图"
  width={800}
  height={450}
  priority={true}
  quality={90}
  loading="eager"
  className="rounded-2xl"
/>
```

**优化措施**：
- 使用 WebP 格式
- 响应式图片（srcset）
- 懒加载（loading="lazy"）
- 占位符（placeholder="blur"）

### 7.2 字体优化

```typescript
// 使用 next/font
import { Space_Grotesk, Source_Han_Sans_SC } from 'next/font/google';

const spaceGrotesk = Space_Grotesk({
  subsets: ['latin'],
  variable: '--font-space-grotesk',
  display: 'swap',
  weight: ['400', '500', '600', '700'],
});

const sourceHanSans = Source_Han_Sans_SC({
  subsets: ['chinese-simplified'],
  variable: '--font-source-han-sans',
  display: 'swap',
  weight: ['300', '400', '500', '700'],
});
```

**优化措施**：
- 使用 next/font 自动优化
- 字体预加载
- 字体子集化（仅包含中文）
- display: swap 避免 FOIT

### 7.3 代码分割

```typescript
// 动态导入
import dynamic from 'next/dynamic';

const Hero = dynamic(() => import('@/components/hero/Hero'), {
  ssr: true,
  loading: () => <HeroSkeleton />,

```

**优化措施**：
- 路由级代码分割
- 组件级动态导入
- 依赖预构建

### 7.4 首屏优化

**目标**：首屏加载 < 1.5s

**措施**：
1. **资源优化**：
   - 图片 WebP + 响应式
   - 字体子集化
   - CSS/JS 压缩

2. **加载策略**：
   - 首屏资源优先加载
   - 非首屏资源懒加载
   - 关键 CSS 内联

3. **渲染优化**：

## 八、部署方案

### 8.1 Vercel 部署

**步骤**：

1. **推送代码到 GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/your-username/selfweb.git
   git push -u origin main
   ```

2. **导入到 Vercel**
   - 访问 https://vercel.com
   - 点击 "Add New Project"
   - 选择 GitHub 仓库
   - 配置环境变量

3. **配置环境变量**
   ```bash
   # Supabase
   NEXT_PUBLIC_SUPABASE_URL=your-supabase-url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
   
   # 其他
   NEXT_PUBLIC_SITE_URL=https://your-website.com
   ```

4. **部署**
   - 点击 "Deploy"
   - 等待构建完成
   - 访问预览 URL

### 8.2 自定义域名

**步骤**：

1. **添加域名**
   - Vercel Dashboard → Settings → Domains
   - 添加自定义域名

2. **配置 DNS**
   - 添加 CNAME 记录
   - 等待 DNS 生效

3. **启用 HTTPS**
   - Vercel 自动提供 HTTPS
   - 配置重定向（HTTP → HTTPS）

### 8.3 CI/CD

**自动部署**：
- 推送到 main 分支 → 自动部署
- 推送到 preview 分支 → 自动预览

**回滚**：
- Vercel Dashboard → Deployments → Rollback

### 10.1 Supabase CLI

**安装**：
```bash
npm install -g supabase
# 或
brew install supabase/tap/supabase
```

**登录**：
```bash
supabase login
```

**链接项目**：
```bash
supabase link --project-ref your-project-ref
```

**生成类型**：
```bash
supabase gen types types --linked
```

### 10.2 数据库迁移

**创建迁移**：
```bash
supabase migration new create_projects_table
```

**应用迁移**：
```bash
supabase migration up
```

**回滚迁移**：
```bash
supabase migration down
```

## 十五、附录

### 15.1 常见问题

**Q: 为什么选择 Next.js 而不是 Gatsby？**
A: Next.js 的 App Router 和 SSR/SSG 混合渲染更适合内容驱动的网站，且开发体验更好。

**Q: 为什么选择 Supabase 而不是 Firebase？**
A: Supabase 基于 PostgreSQL，更适合复杂查询和数据分析，且开源可自托管。

**Q: 如何处理大量图片？**
A: 使用 Next.js Image 组件自动优化，配合 Cloudinary 或 Vercel Images。
